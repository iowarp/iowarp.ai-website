Quick Start Tutorial

Get IOWarp running with Docker in 5 minutes. This tutorial will have you running the IOWarp runtime with buffering services.

Prerequisites​

  • Docker installed (Install Docker)
  • Basic command line familiarity

Pull and Run​

Step 1: Pull the IOWarp Docker Image​

docker pull iowarp/iowarp:latest

Step 2: Run the Container​

docker run -d -p 5555:5555 --name iowarp iowarp/iowarp:latest

What these flags mean:

  • -d - Runs container in background (detached mode)
  • -p 5555:5555 - Exposes IOWarp runtime port
  • --name iowarp - Gives the container a friendly name

Step 3: Verify It's Running​

docker ps

You should see the iowarp container in the running containers list.

Configuration (Optional)​

The default configuration uses RAM-based storage. For custom configurations, use Docker Compose:

cd docker/user
docker-compose up -d

This mounts a custom wrp_conf.yaml configuration file that controls the IOWarp runtime and CTE (Context Transfer Engine) behavior.

Storage Configuration​

The most important parameter is the storage configuration, which defines where and how data is buffered:

# Storage block device configuration
storage:
  # RAM-based storage tier
  - path: "ram::cte_ram_tier1"
    bdev_type: "ram"
    capacity_limit: "16GB"
    score: 0.0  # Tier priority (0.0 = lowest, only one device)

  # Example: Add NVMe tier
  # - path: "/dev/nvme0n1"
  #   bdev_type: "nvme"
  #   capacity_limit: "500GB"
  #   score: 0.5

Key parameters:

  • path - Where data is buffered
  • bdev_type - Storage type (ram or nvme)
  • capacity_limit - Maximum storage size
  • score - Tier priority (higher score = higher priority)

Performance Parameters​

Other configuration parameters primarily affect performance:

  • targets.neighborhood - Number of nodes in the cluster
  • targets.poll_period_ms - How often to rescan targets for statistics
  • dpe.dpe_type - Data placement strategy:
    • "random" - Random placement
    • "round_robin" - Round-robin distribution
    • "max_bw" - Maximum bandwidth optimization

See the Configuration Reference for detailed documentation.

Run a Benchmark (Optional)​

The IOWarp distribution includes benchmark tools to test performance.

Run Default Benchmark​

cd demos/benchmark
docker-compose up

This runs a Put test with default parameters.

Customize Benchmark Parameters​

TEST_CASE=Get IO_SIZE=4m IO_COUNT=1000 docker-compose up

Available parameters:

ParameterDescriptionDefaultExamples
TEST_CASEBenchmark test typePutPut, Get, PutGet
IO_SIZESize of each I/O operation1m4m, 1g, 512k
IO_COUNTNumber of operations1001000, 10000
NUM_PROCSNumber of parallel processes14, 8, 16
DEPTHQueue depth for concurrent ops48, 16, 32

Start Only the Runtime Service​

docker-compose up -d iowarp-runtime

What's Inside the Container​

The iowarp/iowarp:latest image includes:

  • IOWarp Runtime - High-performance I/O runtime
  • CTE - Context Transfer Engine for data buffering
  • Shared Memory - Configured with 8GB for IPC communication
  • Configuration - Default wrp_conf.yaml with RAM-based storage

The container demonstrates:

  • Separate runtime and benchmark services
  • Shared memory configuration (shm_size: 8g)
  • IPC namespace sharing for shared memory access
  • Custom CTE configuration via volume mounts
  • Health checks to ensure runtime readiness
